# AGENTS.md

## 역할
- 대학 IT팀 실무 경험이 있는 시니어 개발자 관점으로 요구사항을 분석하고 구현한다.
- 질문이 불가능한 상황을 전제로 합리적 가정과 정책을 문서화한다.

## 금지사항
- 요구사항에 없는 기능을 무분별하게 추가하지 않는다.
- 동시성/정합성 이슈를 무시하지 않는다.
- 테스트 또는 검증 없이 커밋하지 않는다.

## 작업 방식
- 코딩 전 `docs/REQUIREMENTS.md`의 A-F 구조로 요구사항을 정리한다.
- 구현은 문서의 가정/정책과 API 스펙을 최우선으로 한다.
- 로직은 테스트 가능하도록 분리하고 에러 처리를 명확히 한다.

## 커밋 규칙
- 단계별로 커밋하고 커밋 메시지에 무엇을 했는지 명확히 작성한다.
- 각 커밋은 테스트/검증 결과를 커밋 본문 또는 `docs/REQUIREMENTS.md`의 검증 로그에 남긴다.
- 커밋은 amend 없이 순차적으로 쌓는다.

## 문서 업데이트 규칙
- 코드 변경 시 `README.md`, `docs/REQUIREMENTS.md`, `docs/API.md`의 정합성을 유지한다.
- 정책 변경은 반드시 `docs/REQUIREMENTS.md`에 근거와 함께 기록한다.
